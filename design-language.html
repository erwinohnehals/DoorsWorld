Kairro Design Language

A portable spec distilled from the Kairro task app. Hand this file to Claude Code at the start of a new project and say: “Follow the design language in design-language.html.” Every token, curve, and timing below is copy-paste ready, and the interactive demos on this page run the real animations.

1 · Core principles

  • Calm, warm, quiet — essentially monochrome. Light mode is a warm cream (not pure white); dark mode is a cool gray. The interface is almost entirely surface + ink grays. The single accent hue is held in reserve for rare moments (a toggle's on-state, focus rings) — not for primary buttons, which are solid ink. If you're reaching for the accent, reach for an ink/surface tone first.
  • Semantic tokens only. Components never reference raw hex values. Everything goes through surface / border / ink (content) / accent CSS variables so themes switch with a single class.
  • Motion is the signature. Nothing snaps. Panels pop or grow from their trigger, children cascade in with tight staggers, exits are faster than entrances. All motion is pure CSS (keyframes + inline animation shorthands) — no JS animation libraries, no per-frame tickers.
  • Left-aligned layout. The main content column hugs the sidebar with a small (~24px) gap. Never center the page with mx-auto.
  • Accessible by default. prefers-reduced-motion collapses every animation to instant; 44px minimum touch targets; visible focus rings on non-button elements.

2 · Color

Tokens are defined as space-separated RGB triplets on :root (light) and .dark, consumed as rgb(var(--color-*)) — this keeps Tailwind's / <alpha-value> opacity modifiers working. Dark mode is class-based (darkMode: 'class'), not media-query based. Note the accent changes hue per theme: warm amber in light, warm peach in dark.

A note on the accent: it exists as a token but is used very sparingly — think toggle/switch on-states and focus rings, not fills. Buttons, selected rows, and highlights all use ink/surface tones. Treat the accent as a spice, not a base.

3 · Typography

Inter for everything, with a system-UI fallback stack. Slightly tight tracking gives it the polished feel: -0.01em on body, -0.02em on headings. Headings are semibold (600) — never bold/700 for headings. Body weight 400, medium (500) for buttons and emphasized UI labels.

4 · Motion — the heart of the language

This is what makes the app feel elegant. Read this section carefully before animating anything.

4.1 · The easing curves

NameCurveUse for
STANDARD_EASE ★cubic-bezier(0.625, 0.05, 0, 1)The house curve. Default for every animation: sidebar slide, modals, dropdowns, toasts, list staggers, group reveals. Dramatic start, long luxurious settle.
EXPO_OUTcubic-bezier(0.19, 1, 0.22, 1)Item creation, completion scale — fast arrival, gentle stop.
POWER1_OUTcubic-bezier(0.25, 0.46, 0.45, 0.94)Gentle fades, view-transition entrances.
POWER1_INcubic-bezier(0.55, 0.085, 0.68, 0.53)Exits only (view-out).

★ Export STANDARD_EASE from a single module (e.g. src/lib/easing.ts) and import it — never re-type the numbers.

4.2 · The rules

  • Pure CSS only. Keyframes live in one global stylesheet; JS only measures, sets inline animation/transition shorthands and delays. No GSAP / Framer Motion / per-frame tickers — motion stays smooth while the main thread is busy.
  • Single-endpoint keyframes. Define only from {} for entrances and only to {} for exits. The other end is the element's own base style, so hover/drag/state classes survive the animation and one keyframe serves many components.
  • Exits are faster than entrances — roughly 50–70% of the enter duration. (Dropdown: in 192ms, out 115ms. Modal: in 320ms, out 200ms.)
  • Cascade children. When a container reveals, its rows stagger in with 15–50ms steps, starting before the container finishes (overlap, never sequential). Use animation-fill-mode: backwards so delayed items stay hidden through their delay.
  • Cap stagger totals. Clamp the max delay (sidebar: 420ms open / 180ms close) and cap staggered item counts (~24) so long lists never feel slow. Close staggers run bottom-to-top.
  • Animate transform + opacity only. Translate distances are small: 10–20px rises, 0.95–0.97 scales. Exception: the dropdown 'expand' variant animates width/height fed through CSS custom properties.
  • Keep exiting elements mounted until animationend, with a setTimeout fallback (duration + 100ms) since animationend can be swallowed in backgrounded tabs.
  • Interruptible. Reopening mid-close restarts cleanly: clear the inline animation, force a reflow (el.getBoundingClientRect()), then reassign — an identical shorthand won't restart otherwise. The expand variant re-grows from its current partial size.
  • Respect prefers-reduced-motion in every animation path: skip straight to the end state.

4.4 · The sliding menu highlight (segmented control) ★

A signature move, and the one that makes navigation feel alive: instead of each item toggling its own background on and off, a single highlight pill glides from the previously selected item to the new one. Click across the tabs below — notice how the highlight travels between items rather than blinking from place to place, and how it resizes to fit each label:

How it's built:

  • One pill, not per-item backgrounds. A single absolutely-positioned element lives inside a position: relative track, sitting behind the items (items get a higher z-index and a transparent background). The pill is surface-sunken with a whisper of shadow.
  • It glides via a CSS transition on transform, width, height — 400ms with the house STANDARD_EASE. Position it with translate(activeBtn.offsetLeft, activeBtn.offsetTop) and copy the button's offsetWidth/Height, so it molds to items of different widths.
  • Always transition from the current position — never snap-then-animate. Because it starts wherever it happens to be, rapid clicks retarget it mid-flight and it curves smoothly to the new target.
  • First paint snaps (set transition: none for the initial placement) so the pill doesn't fly in from the corner on mount. Reduced-motion snaps every time.
  • Item labels only cross-fade their color (transition: color 300ms): ink-2 at rest → ink when active, arriving as the pill does. No layout shift, no background flash — just color.
  • Re-measure on layout change with a ResizeObserver on the track (font load, count badges appearing, window/resize) and reposition the pill so it never drifts out of alignment.
// Move the pill to the active item — React or vanilla, same idea.
// `animate = false` on first paint and for reduced-motion (snap into place).
const movePill = (activeBtn, animate) => {
  pill.style.transition = animate
    ? ['transform','width','height'].map(p => `${p} 400ms ${STANDARD_EASE_CSS}`).join(', ')
    : 'none';
  pill.style.transform = `translate(${activeBtn.offsetLeft}px, ${activeBtn.offsetTop}px)`;
  pill.style.width  = `${activeBtn.offsetWidth}px`;
  pill.style.height = `${activeBtn.offsetHeight}px`;
  pill.style.opacity = '1';
};
// Keep it aligned when the track's layout shifts:
new ResizeObserver(() => movePill(currentActiveBtn, false)).observe(track);

/* Generic reveal — rise distance via --rise-y (12/16/20px, −10px for create) */
@keyframes rise-in  { from { opacity: 0; transform: translateY(var(--rise-y, 16px)); } }

/* Dropdown 'pop' + row cascade */
@keyframes dd-pop-in   { from { opacity: 0; transform: translateY(-10px) scale(0.96); } }
@keyframes dd-pop-out  { to   { opacity: 0; transform: translateY(-6px)  scale(0.97); } }
@keyframes dd-item-in  { from { opacity: 0; transform: translateY(14px); } }
@keyframes dd-item-out { to   { opacity: 0; } }

/* Dropdown 'expand' — sizes measured by JS, fed via custom props */
@keyframes dd-expand-in  { from { width: var(--dd-from-w); height: var(--dd-from-h); }
                           to   { width: var(--dd-to-w);   height: var(--dd-to-h); } }
@keyframes dd-expand-out { from { width: var(--dd-from-w); height: var(--dd-from-h); }
                           to   { width: var(--dd-to-w);   height: 0px; } }

/* Views, modal, toast */
@keyframes view-in  { from { opacity: 0; transform: translateX(20px); } }
@keyframes view-out { to   { opacity: 0; transform: translateX(-20px); } }
@keyframes modal-backdrop-in  { from { opacity: 0; } }
@keyframes modal-backdrop-out { to   { opacity: 0; } }
@keyframes modal-panel-in  { from { opacity: 0; transform: translateY(16px) scale(0.96); } }
@keyframes modal-panel-out { to   { opacity: 0; transform: translateY(10px) scale(0.97); } }
@keyframes toast-in  { from { opacity: 0; transform: translateY(30px) scale(0.95); } }
@keyframes toast-out { to   { opacity: 0; transform: translateY(50px) scale(0.95); } }

/* Button press — apply with `linear`; easing lives inside */
@keyframes btn-press {
  0%   { transform: scale(1);    animation-timing-function: cubic-bezier(0.55,0.085,0.68,0.53); }
  50%  { transform: scale(0.95); animation-timing-function: cubic-bezier(0.25,0.46,0.45,0.94); }
  100% { transform: scale(1); }
}

/* Apply from JS/components, e.g.: */

5 · Buttons & interactive elements

  • Radius: rounded-lg (8px). Font-weight 500. Sizes: sm px-3 py-1.5 text-sm, md px-4 py-2 text-base, lg px-6 py-3 text-lg.
  • Variants: primary solid ink bg / surface text (high-contrast, deliberately not the accent), hover to ink-2; secondary surface-2 bg with border, hover surface-3; ghost transparent, ink-2 text, hover surface-3; danger error red. The accent is not a button color here.
  • Press feedback: active:scale-95 plus the btn-press keyframe fired on click. Disabled: opacity-50 cursor-not-allowed.
  • Touch: 44×44px minimum targets on all buttons/links; -webkit-tap-highlight-color: transparent.
  • All interactive elements get the global 150ms transition on color/background/border/transform/shadow.
  • Focus: 2px accent :focus-visible outline with 2px offset — suppressed on buttons (press feedback covers it), kept for inputs/links.

6 · Borders, radii, elevation

ElementRadiusNotes
Buttons, inputs, menu rows, list rows8px rounded-lgThe workhorse radius — when unsure, use this
Cards, panels, dropdown panels12px rounded-xl1px border token, surface-2 fill
Modals, hero cards16px rounded-2xlWith shadow-2xl
Chips, tags, avatars, dotsrounded-fullPills for tags and counters
Tiny inline elements (kbd, badges)6px rounded-md
  • Borders do the separating, not shadows. Flat 1px border-token lines everywhere; border-strong for emphasis (toasts, dropdown panels).
  • Shadows are reserved for genuinely floating layers: dropdowns/toasts shadow-xl, modals shadow-2xl, subtle card-hover lift (translateY(-2px) + soft shadow, 200ms).
  • Glass effect for overlaid bars: surface-2 at 80% + backdrop-blur(12px).

7 · Icons

  • Library: lucide-react — thin, consistent stroke icons. Never mix icon families in one surface.
  • Default size 16px (w-4 h-4) inline with text; 20px (w-5 h-5) for primary nav / standalone icon buttons.
  • Icons inherit text color: ink-2/ink-3 at rest, ink on hover. Color only for status (success/warning/error) or the accent.
  • Icon + label spacing: gap-2 (8px). Icon-only buttons are flex-centered with the 44px touch minimum.

8 · Micro-details that complete the feel

  • Custom scrollbars: 8px wide, surface-3 track, border-strong rounded thumb.
  • Text selection: blue #3B82F6 with white text (both themes).
  • Inputs: no default focus ring/shadow (outline: none on focus) — rely on border-color changes and the container's affordances.
  • Empty states and loading spinners stay quiet: ink-3 text, simple border-spin spinner (border-2 border-current border-t-transparent rounded-full animate-spin).
  • Toasts: bottom-right, surface-2 with border-strong, a thin progress bar (border-strong fill) that drains linearly over the toast's lifetime.
  • kbd hints: surface-3 fill, border-strong with a thicker bottom edge.

9 · Quick-start checklist for the agent

  • ☐ Copy the CSS variable block + Tailwind token config (§2). Use only semantic tokens in components.
  • ☐ Inter font stack, tracking −0.01em body / −0.02em headings, headings at 600 (§3).
  • ☐ Create src/lib/easing.ts exporting STANDARD_EASE_CSS = 'cubic-bezier(0.625, 0.05, 0, 1)'.
  • ☐ Copy the keyframe library into the global stylesheet (§4.6). Pure CSS animation only — no animation libraries.
  • ☐ Every panel/menu/modal: animated enter and faster exit, children cascading with backwards-fill staggers, reduced-motion respected, kept mounted until animationend (+timeout fallback).
  • ☐ Segmented controls / tab bars use the sliding highlight pill — one pill, glides 400ms from its current position, snaps on first paint (§4.4).
  • ☐ Keep the accent nearly unused: monochrome ink/surface everywhere; accent only for toggle on-states and focus rings, never for primary buttons (§2).
  • ☐ Buttons: rounded-lg, 500 weight, the four variants (primary = solid ink, not accent), active:scale-95 + btn-press (§5).
  • ☐ Radii ladder 8/12/16/full; borders over shadows (§6).
  • ☐ lucide-react icons, 16px inline, color from text (§7).
  • ☐ Left-aligned layout — never mx-auto-centered pages.
